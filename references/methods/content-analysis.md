# Content Analysis

## Overview

Content Analysis systematically categorizes and quantifies communication content. It bridges qualitative and quantitative approaches: you can count occurrences (quantitative CA) or interpret meaning patterns (qualitative CA). Useful when analyzing survey open-ends, system logs, social media, or any structured text data.

## Types

| Type | Approach | When to Use |
|------|----------|-------------|
| Conceptual | Count frequency of specific terms/concepts | Quick mapping of topical coverage |
| Relational | Analyze relationships between concepts | Understanding how concepts co-occur |
| Qualitative | Systematic coding of latent content | Deeper meaning beyond surface terms |
| Summative | Combines quantitative counting with qualitative interpretation | Most HCI-friendly approach |

## Procedure

### 1. Define the Unit of Analysis
- Word, phrase, sentence, paragraph, or entire response.
- Choose based on research question and data granularity needed.

### 2. Develop a Coding Scheme
**For deductive CA (theory-driven):**
- Derive categories from existing literature, theories, or frameworks.
- Define inclusion/exclusion criteria for each category.
- Pilot test on a subset; refine.

**For inductive CA (data-driven):**
- Start with open coding; iteratively refine categories.
- Build the scheme through engagement with data.

**Category requirements:**
- Mutually exclusive (one unit should fit one category)
- Exhaustive (every unit should fit some category)
- Independent coding (coder assignment doesn't depend on other units)
- Objectively applicable (different coders reach same classification)

### 3. Code the Data
- Apply coding scheme systematically across all data.
- Record both the category assignment and the evidence.
- For inter-coder reliability: have at least 2 coders independently code a subset (typically 20-30%).

### 4. Analyze Patterns
**Quantitative:**
- Frequency counts and percentages per category
- Cross-tabulations (category × participant group)
- Statistical tests: Chi-square, Fisher's exact test for category differences

**Qualitative:**
- Examine patterns in how categories manifest
- Look for relationships between categories
- Consider context, tone, and nuance

### 5. Report
- Present the coding scheme (what and how was coded).
- Report reliability metrics (Cohen's Kappa, percent agreement).
- Show frequency tables with qualitative illustrations.
- Discuss implications, limitations.

## HCI Applications

| Use Case | What to Analyze |
|----------|----------------|
| Survey open-ended responses | Categorize reasons, suggestions, complaints |
| System logs / chat data | Identify interaction patterns and intents |
| Social media content | Track sentiment, topics, community behavior |
| Interview transcripts | Systematically count strategic topics across participants |
| Design feedback | Categorize usability issues by type and severity |

## When to Choose Content Analysis

- You have a large volume of textual data (more suitable than deep TA for 100+ responses).
- You need both frequency information AND qualitative depth.
- You want to compare across groups or conditions.
- You have a pre-existing framework to apply deductively.
