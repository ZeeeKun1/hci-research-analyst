# Affinity Diagramming

## Overview

Affinity Diagramming (AD) is a bottom-up clustering technique widely used in HCI and design research (particularly at companies like Intel, as documented by Beyer & Holtzblatt in Contextual Design). It organizes large volumes of qualitative data into a hierarchical structure through collaborative sorting.

## Core Idea

Individual data points (notes/quotes/observations) are grouped based on inherent similarity, building a bottom-up hierarchy: notes → blue labels (sub-themes) → pink labels (themes) → green labels (super-themes).

## Procedure

### 1. Extract Data Points
- Break qualitative data (interviews, observations, field notes) into individual notes.
- Each note should be:
  - Self-contained and interpretable without context
  - At a consistent granularity (not too broad, not too specific)
  - Written in the participant's voice or clearly attributed

### 2. Spread and Sort
- Lay out all notes visibly.
- Sort notes into groups based on affinity (natural similarity).
- Sort silently at first to avoid anchoring bias.
- Key rule: No pre-defined categories. Groups emerge from the data.

### 3. Label Groups
- For each group, write a label that captures the shared essence.
- Labels should be:
  - Higher-level abstraction than the notes
  - Written as insights, not just topics
  - Testable against the notes (every note in the group should fit)
- Sub-layers: Individual notes → Sub-group labels → Group labels → Super-group labels.

### 4. Build the Hierarchy
- Organize labels into a tree structure.
- Walk the tree from bottom to top, verifying coherence at each level.
- Review: Are groups distinct? Are labels accurate? Are there orphan notes?

### 5. Extract Insights
- For each high-level group, synthesize:
  - What is the pattern?
  - Why does it matter?
  - What does it suggest for design or theory?
- Use the hierarchy to tell a coherent story.

## HCI-Specific Guidance

- **Contextual Design tradition**: Affinity diagrams often accompany Contextual Inquiry data. Notes capture users' work practices, pain points, and strategies.
- **Design research**: The output directly informs design requirements and personas.
- **Academic analysis**: Can be used as a precursor to more formal thematic analysis, or as a standalone method when rapid insight extraction is needed.
- **Collaborative AD**: When multiple researchers sort, note disagreements — they often reveal important boundary cases.

## Comparison with Thematic Analysis

| Dimension | Affinity Diagramming | Thematic Analysis |
|-----------|---------------------|-------------------|
| Process | Physical/digital sorting | Systematic coding & theme development |
| Output | Hierarchical cluster map | Themes with analytic narrative |
| Rigor | Good inter-rater reliability through sorting | Explicit phase-by-phase methodology |
| Best for | Rapid synthesis, design research | Formal academic reporting |
| Theory use | Light; primarily data-driven | Can be theory-driven or data-driven |

## When to Choose Affinity Diagramming

- Large volume of short data points (e.g., many short interviews, observation notes).
- Need rapid synthesis for design feedback.
- Collaborative analysis with multiple researchers.
- Data is relatively concrete and behavioral (less suitable for abstract/reflective data).
