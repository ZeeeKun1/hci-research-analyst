# Affordance Theory

## Overview

Affordance, originally from ecological psychology (Gibson, 1977), refers to the action possibilities that an environment offers an actor. In HCI, Norman (1988) popularized the concept to describe perceived and actual design properties that suggest interaction.

## Core Constructs

### Gibson's Original
- **Affordance**: A relational property between an organism and its environment. A chair affords sitting for humans, but not for a snake.
- Affordances exist whether perceived or not. They are objective properties of the environment relative to the actor.
- Can be positive (afford action) or negative (prevent action).

### Norman's HCI Adaptation
- **Perceived Affordance**: What the user perceives they can do (may differ from actual affordance).
- **Signifiers**: Visual cues that indicate where and how to act (labels, icons, highlights).
- **Constraints**: Limitations on action possibilities (physical, logical, cultural).
- **Mappings**: Relationship between controls and their effects (natural mapping reduces learning).
- **Feedback**: Information about the result of an action.

### Gaver's Technology Affordances
- Distinguishes between: perceptible, hidden, false, and rejectable affordances.
- Highlights that complex technologies create ambiguous affordances.

## Key Predictions

1. Well-designed perceived affordances reduce learning time and errors.
2. Mismatch between actual and perceived affordances causes user confusion.
3. Cultural conventions affect affordance perception (e.g., scroll direction).
4. Affordances are discovered through exploration — first-time users rely on them most.
5. Multimodal interfaces can create competing affordances → need clear prioritization.

## HCI Applications

| Context | Application |
|---------|-------------|
| Interaction design | Design clear perceived affordances through visual/physical cues |
| Tangible interfaces | Physical form determines inherent affordances |
| Accessibility | Affordances differ by ability — design for diverse actor capabilities |
| Mobile design | Touch affordances (tap, swipe, pinch) depend on finger size and screen |
| Onboarding | Reduce need for instructions by making affordances self-evident |

## When Findings Map to Affordance Theory

- Users confused about how to interact → Perceived affordance mismatch
- Users discover unintended uses → Hidden affordance
- Users expect action but it doesn't work → False affordance
- Users don't notice a feature → Affordance not perceptible
- Users from different cultures interact differently → Cultural convention effect on affordance
