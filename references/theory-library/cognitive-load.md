# Cognitive Load Theory

## Overview

Cognitive Load Theory (CLT), developed by John Sweller (1988, 1994), explains how the limited capacity of working memory affects learning and task performance. It distinguishes between beneficial and detrimental load imposed by instructional/task design.

## Core Constructs

- **Working Memory Capacity**: Limited to ~4-7 elements simultaneously. Central bottleneck in processing.
- **Intrinsic Load**: Inherent difficulty of the task/content. Cannot be eliminated, only managed by scaffolding.
- **Extraneous Load**: Load imposed by poor design or presentation. Always harmful. Target for reduction.
- **Germane Load**: Cognitive effort devoted to schema construction and automation. Always beneficial. Target for promotion.
- **Schema**: Mental model or knowledge structure. Automation of schemas frees working memory.

## Key Predictions

1. Performance degrades when total cognitive load exceeds working memory capacity.
2. Splitting attention across multiple sources increases extraneous load (split-attention effect).
3. Redundant information increases extraneous load (redundancy effect).
4. Worked examples are more effective than problem-solving for novices (worked-example effect).
5. Expertise reversal: techniques effective for novices may hinder experts (expertise reversal effect).
6. Modality principle: dual-channel presentation (visual + auditory) can increase effective capacity.

## HCI Applications

| Context | How CLT Informs Design |
|---------|----------------------|
| Information architecture | Reduce extraneous load by minimizing unnecessary navigation steps |
| Dashboard design | Present only relevant information; avoid information overload |
| Learning systems | Scaffold complex tasks; use progressive disclosure |
| Onboarding | Use worked examples and guided tours rather than free exploration |
| Multitasking interfaces | Monitor cognitive demands; provide focus modes |
| Complex systems | Support schema formation through consistent patterns |

## Measuring Cognitive Load

- **Subjective**: NASA-TLX (task load index), Paas scale (simple 9-point)
- **Objective**: Pupillometry, EEG, dual-task performance, response time
- **Behavioral**: Error rates, completion time patterns (sudden jumps indicate overload)
