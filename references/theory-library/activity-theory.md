# Activity Theory

## Overview

Activity Theory (AT), rooted in Russian psychology (Leontiev, Vygotsky), analyzes human behavior as purposeful, tool-mediated activity situated in a social context. Widely used in HCI for understanding how technology mediates work practices.

## Core Constructs — Engeström's Activity System

An activity system consists of six nodes:
1. **Subject**: The person or group being studied.
2. **Object**: The objective/motive of the activity (what the subject is trying to achieve).
3. **Tools/Mediating Artifacts**: Physical or psychological tools used (technology, language, methods).
4. **Rules**: Norms, regulations, conventions governing the activity.
5. **Community**: The social environment and other stakeholders.
6. **Division of Labor**: How tasks are distributed among participants.

Relationships: Subject ↔ Object (driven by motive); all other nodes mediate this relationship. Tensions between nodes are the primary unit of analysis.

## Key Predictions

1. Introducing new technology creates tensions in the activity system — these are the engine of change.
2. Contradictions (tensions) operate at multiple levels: primary (within a node), secondary (between nodes), tertiary (between old and new activity systems), quaternary (between the activity and its surrounding culture).
3. Tool use transforms both the object and the subject — technology is not neutral.
4. The same tool may mediate different activities differently depending on context.
5. Activity systems evolve through expansive learning cycles (Engeström).

## HCI Applications

| Context | How AT Informs Analysis |
|---------|------------------------|
| CSCW | Understanding how collaborative tools reshape work |
| Workflow analysis | Identifying contradictions when introducing new systems |
| Organizational tech | Analyzing rules, community, and division of labor |
| Ubiquitous computing | Understanding how embedded tools mediate everyday activity |
| Design research | Using contradictions as design opportunities |

## When Findings Map to AT

- Users mention workflow disruptions → Contradiction between old/new tools
- Users develop workarounds → Primary contradiction (tool doesn't fit object)
- Organizational resistance → Tension between community and rules
- Role shifts due to technology → Division of labor contradiction
- New practices emerge → Expansive learning cycle
