# Distributed Cognition

## Overview

Distributed Cognition (DCog), developed by Edwin Hutchins (1995) from his studies of ship navigation, proposes that cognitive processes are not confined to the individual mind but are distributed across people, tools, and the environment. A unit of analysis is the cognitive system, not the individual.

## Core Constructs

- **Cognitive System**: The functional system comprising people, artifacts, and environment working together to accomplish a task.
- **Propagating Representations**: Information transforms as it moves between internal (mental) and external (artifacts) representations.
- **Mediating Artifacts**: Tools that represent, transform, and transmit information (maps, displays, checklists, software).
- **Coordination**: Distributed cognitive processes are coordinated through social interaction, shared artifacts, and representational states.

### Key Principles
1. Cognition is a property of systems, not individuals.
2. Representational state is the fundamental unit of analysis.
3. Cognitive processes propagate through a system of representational media.
4. Functional organization matters more than physical location.

## Key Predictions

1. Removing or altering a mediating artifact changes the cognitive process, even if the task is the "same."
2. System performance depends on the fit between representational formats and cognitive operations.
3. Errors arise from misalignment between representations at different stages of propagation.
4. Expert performance depends on coordination across distributed elements, not individual skill alone.
5. Changes to one part of the system (e.g., new software) can create cascading effects.

## HCI Applications

| Context | Application |
|---------|-------------|
| Collaborative systems | Analyze how information flows between people and tools |
| Air traffic control, medical, nuclear | Study high-stakes distributed cognitive systems |
| Dashboard/analytics design | Optimize representational formats for cognitive operations |
| Ubiquitous computing | Understand cognition across devices and environments |
| Team cognition | Analyze shared mental models through shared artifacts |

## When Findings Map to Distributed Cognition

- Users coordinate across multiple tools/screens → Mediating artifact analysis
- Errors occur at handoff points → Representation propagation breakdown
- Performance improves when a tool is redesigned → Representational alignment
- Team expertise ≠ individual expertise → Distributed cognitive process
- Users create external representations (notes, drawings, annotations) → Cognitive offloading
